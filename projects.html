Matrícula | 2025

Link to the project's documentation can be found here!

This is probably the project that I'm the most proud of (so far). I knew I wanted to develop this project for a while before sitting down and developing it.

I won't go into extensive detail about the project iself, as a thorough documentation can be found on the link above (which I encourage everyone to read), I will focus more on my thought process for building it.

Stack

  • Frontend: React
  • Backend: Django

Matrícula overview

  • Matrícula has the explicit goal of being an improved version of the current system of University of Puerto Rico, which is terminal-based.
  • Matrícula was built as a "rough sketch" or showcase of what a web-based enrollment system for the university could look like.
  • Matrícula took about six months to fully develop, from requirement gathering to final product.

Matrícula accomplishments

  • Improving user experience all around
    • Developing a web-based enrollment system, as opossed to having to use a terminal
    • Making navigation across the platform as straight-forward as possible
    • Adding quality-of-life features that were not present in the original system, based on students' feedback
  • Showing what's possible
    • Showcased that it's possible to have a web-based enrollment system by myself (solo-development)
    • Addressed all major issues with the platform, which all students can agree on

ProfeStocks | 2024

Link to the live project can be found here!

This is the first complete project I ever did! The making of this project is what solidified that I wanted to Software Engineer. Yes I built it with PHP because that was the only language I knew at the time.

  • This project wouldn't have been possible without the help of Sebastián Sterling, who did all the frontend for this site.

This project was made open source in 2025, and the full project source code can be found here! Since this project is open source, I won't go into extensive detail here either.

Stack

  • Frontend: Vanilla PHP + CSS
  • Backend: PHP

ProfeStocks overview

  • ProfeStocks has the explicit goal of improving upon the already existing professor rating website, used by students of University of Puerto Rico.
    • Link to that original site can be found here
  • ProfeStocks was to also be used as refresher of the original site, as that site hadn't be maintained for a few years at least.
  • ProfeStocks took about a year to fully develop, from requirement gathering to final product.

ProfeStocks accomplishments

  • Cleaning the site overall
    • Looking at the source site, we opted for a cleaner, more organized, user-friendly, look.
    • We realized that the data hadn't been updated in years, so we decided to refresh the data with the most recent professor and courses available as of 2024-2025.
  • Adding missing features
    • The website was lacking some features we considered essential, they are listed below.
    • Every comment posted on the site has the date it was posted in.
      • This is because an opinion of a professor can easily become outdated.
    • We matched/mapped professors to every course they may teach.
      • This was done to ensure students are rating the correct professor, as professors with the same name exist.
  • ProfeStocks became a sensation across UPR Rio Piedras campus, and is where students default to check professor reviews now.
